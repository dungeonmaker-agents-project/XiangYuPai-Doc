# 05-关注列表页面

## 一、页面概述

- **路由路径**: `/user/following` 或 `/users/{userId}/following`
- **页面功能**: 查看关注列表，支持搜索和取消关注操作
- **用户角色**: 已登录用户
- **页面类型**: 列表页面

## 二、页面结构

### 2.1 顶部导航栏

```
[←]  关注列表                        [🔍]
```
- 返回按钮：左侧箭头图标
- 标题："关注列表"（居中，18pt）
- 搜索按钮：右侧放大镜图标
- 高度：44px
- 底部分割线：1px 浅灰色

### 2.2 搜索栏（点击搜索按钮后展开）

```
┌────────────────────────────────────────┐
│  [🔍 搜索关注用户昵称_______________ ⊗] │
└────────────────────────────────────────┘
```

**搜索输入框:**
- 占位符："搜索关注用户昵称"
- 输入类型：`text`
- 实时搜索：输入后延迟500ms自动搜索
- 清空按钮：有内容时显示

### 2.3 关注列表

**列表项:**
```
┌────────────────────────────────────────┐
│  [头像] 昵称                  [已关注]  │
│        简介文字...                      │
│        📍 广东省广州市  📏 175cm        │
└────────────────────────────────────────┘
```

**头像:**
- 圆形头像，48x48px
- 点击跳转到用户主页
- 右下角显示在线状态指示器

**用户信息:**
- 昵称：粗体，16pt
- 简介：常规字体，12pt，灰色
- 居住地、身高：图标 + 文字，12pt，灰色

**关注按钮:**
- **已关注**：灰色边框按钮，文字"已关注"，宽度80px
- **互相关注**：紫色边框按钮，文字"互相关注"，宽度80px
- 高度：32px，圆角：16px
- 点击弹出确认对话框

**分割线:**
- 每个列表项之间有1px浅灰色分割线

### 2.4 空状态

**无关注时:**
```
┌────────────────────────────────────────┐
│           🌟                            │
│       还没有关注任何人                  │
│    去发现感兴趣的人和内容吧！           │

```

**搜索无结果时:**
```
┌────────────────────────────────────────┐
│           🔍                            │
│       没有找到相关用户                  │
│      试试其他关键词吧                   │
└────────────────────────────────────────┘
```

### 2.5 加载更多

**上拉加载:**
- 滑动到底部自动加载下一页
- 显示"加载中..."
- 全部加载完显示"没有更多了"

## 三、接口列表

### 3.1 获取关注列表

**接口:** `GET /api/users/following`

**触发时机:** 页面加载时、上拉加载更多

**请求头:**
```
Authorization: Bearer <token>
```

**请求参数:**
```typescript
{
  page: number;      // 页码，从1开始
  pageSize: number;  // 每页数量，默认20
}
```

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: {
    following: Array<{
      userId: string;
      nickname: string;
      avatar: string;
      bio?: string;
      residence?: string;
      height?: number;
      isOnline: boolean;
      followStatus: "following" | "mutual";  // 已关注或互相关注
      followedAt: string;  // 我关注TA的时间
    }>;
    total: number;
    hasMore: boolean;
  };
}
```

### 3.2 搜索关注列表

**接口:** `GET /api/users/following/search`

**触发时机:** 搜索框输入内容后（防抖500ms）

**请求头:**
```
Authorization: Bearer <token>
```

**请求参数:**
```typescript
{
  keyword: string;   // 搜索关键词
  page: number;
  pageSize: number;
}
```

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: {
    following: Array<{
      userId: string;
      nickname: string;
      avatar: string;
      bio?: string;
      residence?: string;
      height?: number;
      isOnline: boolean;
      followStatus: "following" | "mutual";
      followedAt: string;
    }>;
    total: number;
    hasMore: boolean;
  };
}
```

### 3.3 取消关注

**接口:** `DELETE /api/users/follow/{targetUserId}`

**触发时机:** 点击"已关注"或"互相关注"按钮，确认后

**请求头:**
```
Authorization: Bearer <token>
```

**路径参数:**
- `targetUserId`: 要取消关注的用户ID

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: {
    followStatus: "none";
  };
}
```

## 四、数据流

### 页面加载流程
```
用户进入关注列表页面
  ↓
GET /api/users/following (page=1)
  ↓
显示关注列表
  ├─ 有数据: 显示列表
  └─ 无数据: 显示空状态
```

### 搜索流程
```
用户点击搜索按钮
  ↓
展开搜索框，聚焦
  ↓
用户输入关键词
  ↓
防抖500ms
  ↓
GET /api/users/following/search (keyword, page=1)
  ↓
显示搜索结果
  ├─ 有结果: 显示列表
  └─ 无结果: 显示"没有找到相关用户"
```

### 取消关注流程
```
用户点击"已关注"或"互相关注"按钮
  ↓
弹出确认对话框
  - 标题: "取消关注"
  - 内容: "确定取消关注【昵称】吗？"
  - 按钮: "取消" / "确定"
  ↓
用户点击"确定"
  ↓
DELETE /api/users/follow/{targetUserId}
  ↓
成功:
  - 从列表中移除该用户
  - 显示Toast "已取消关注"
  - 更新关注总数
失败:
  - 显示错误提示
```

### 分页加载流程
```
用户滑动到列表底部
  ↓
检查是否还有更多数据 (hasMore)
  ↓
如果有:
  - 调用接口加载下一页 (page = currentPage + 1)
  - 追加数据到列表末尾
如果没有:
  - 显示"没有更多了"
```

## 五、状态管理

interface FollowingListPageState {
  // 列表数据
  following: {
    data: FollowingUser[];

  };

  // 搜索状态

  height?: number;

}

## 六、前端验证

### 搜索关键词验证
```typescript
const validateSearchKeyword = (keyword: string) => {
  // 去除首尾空格
  const trimmed = keyword.trim();

  // 最小长度检查
  if (trimmed.length === 0) {
    return { valid: false, error: '请输入搜索关键词' };
  }

  // 最大长度检查
  if (trimmed.length > 50) {
    return { valid: false, error: '搜索关键词不能超过50字符' };
  }

  return { valid: true, keyword: trimmed };
}
```

## 七、交互行为

### 7.1 页面加载

**进入页面:**
1. 显示加载状态
2. 调用关注列表接口
3. 成功：显示列表
4. 失败：显示错误提示和重试按钮

### 7.2 搜索交互

**点击搜索按钮:**
1. 展开搜索框
2. 自动聚焦到搜索输入框
3. 进入搜索模式

**输入搜索关键词:**
1. 实时显示输入内容
2. 防抖500ms后自动搜索
3. 显示搜索加载状态
4. 显示搜索结果

**清空搜索:**
1. 点击清空按钮（⊗）
2. 清空搜索框
3. 显示完整关注列表
4. 退出搜索模式

**返回操作（搜索模式）:**
1. 点击返回按钮
2. 如果在搜索模式：
   - 清空搜索
   - 退出搜索模式
   - 显示完整列表
3. 如果不在搜索模式：
   - 返回上一页

### 7.3 列表交互

**点击列表项（头像或昵称）:**
1. 跳转到用户主页
2. 路由: `/users/{userId}/profile`
3. 传递userId

**点击关注按钮:**
1. 弹出取消关注确认对话框
2. 对话框内容：
   - 标题："取消关注"
   - 内容："确定取消关注【昵称】吗？"
   - 提示："取消后将不再接收TA的动态更新"
   - 按钮："取消" / "确定"
3. 点击"确定"：
   - 调用取消关注接口
   - 显示取消关注加载状态
   - 成功：从列表中移除该用户，显示Toast
   - 失败：显示错误提示

**下拉刷新:**
1. 下拉列表顶部
2. 显示刷新图标和文字
3. 释放触发刷新
4. 重新加载第1页数据
5. 清除旧数据
6. 显示刷新成功提示

**上拉加载更多:**
1. 滑动到列表底部（距底部100px触发）
2. 检查hasMore状态
3. 如果hasMore = true:
   - 显示"加载中..."
   - 调用接口加载下一页
   - 追加数据到列表
4. 如果hasMore = false:
   - 显示"没有更多了"

### 7.4 取消关注确认对话框

**对话框结构:**
```
┌────────────────────────────────────────┐
│        取消关注                         │
│                                         │
│  确定取消关注【小明】吗？               │
│  取消后将不再接收TA的动态更新           │
│                                         │
│     [取消]           [确定]             │
└────────────────────────────────────────┘
```

**按钮:**
- 取消按钮：灰色边框，关闭对话框
- 确定按钮：红色实心，执行取消关注

**取消关注成功:**
1. 关闭对话框
2. 从列表中移除该用户（带淡出动画）
3. 显示Toast："已取消关注"
4. 更新关注总数

**取消关注失败:**
1. 关闭对话框
2. 显示Toast错误提示
3. 保持列表不变

### 7.5 空状态交互

**点击"去发现"按钮:**
1. 跳转到发现页面
2. 路由: `/discover`
3. 可以浏览推荐用户和内容

### 7.6 错误处理

**网络错误:**
1. 显示错误提示："网络异常，请稍后重试"
2. 显示重试按钮
3. 点击重试重新加载

**Token过期:**
1. 提示："登录已过期，请重新登录"
2. 跳转到登录页

## 十一、与粉丝列表的区别

### 功能差异

| 功能 | 粉丝列表 | 关注列表 |
|------|---------|---------|
| 关注操作 | 可以关注/取消关注 | 只能取消关注 |
| 按钮状态 | 未关注/已关注/互相关注 | 已关注/互相关注 |
| 操作确认 | 取消关注需要确认 | 取消关注需要确认 |
| 空状态 | "还没有粉丝" | "还没有关注任何人" + 去发现按钮 |
| 时间字段 | followedAt（TA关注我的时间） | followedAt（我关注TA的时间） |

### UI差异

**粉丝列表:**
- 关注按钮可以是三种状态
- 点击关注立即执行
- 点击取消关注需要确认

**关注列表:**
- 关注按钮只有两种状态（已关注/互相关注）
- 点击任何状态都需要确认
- 提供"去发现"引导

### 业务逻辑差异

**粉丝列表:**
- 显示关注我的人
- 我可以选择回关
- 互相关注后变成朋友关系

**关注列表:**
- 显示我关注的人
- 我可以选择取消关注
- 取消后对方仍可能是我的粉丝

## 十二、优化建议

### 性能优化

**列表虚拟滚动:**
- 当关注数量超过1000时，使用虚拟滚动
- 只渲染可见区域的列表项

**图片懒加载:**
- 头像图片使用懒加载
- 只加载可见区域的图片

**防抖优化:**
- 搜索输入防抖500ms
- 避免频繁请求

### 用户体验优化

**动画效果:**
- 取消关注后列表项淡出动画
- 列表项高度平滑过渡

**确认对话框:**
- 提供更友好的确认提示
- 说明取消关注的后果

**批量操作（可选）:**
- 长按进入批量模式
- 可以批量取消关注
- 适用于清理关注列表

**分组显示（可选）:**
- 按关注时间分组
- 今天、本周、本月、更早
- 方便查找最近关注的用户
