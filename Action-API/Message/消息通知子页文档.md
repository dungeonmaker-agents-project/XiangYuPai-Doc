# 消息通知子页文档

## 概述
本文档定义消息模块的四个通知子页面的完整实现,包括赞和收藏、评论、粉丝、系统通知的UI布局、交互行为、前端逻辑和API接口规范。这四个页面共享相似的列表结构,但具有各自独特的数据展示和交互逻辑。

**本文档范围:**
- 赞和收藏通知页
- 评论通知页
- 粉丝通知页
- 系统通知页

**相关文档:**
- 消息页面文档 (会话列表和聊天功能)
- 用户关系文档 (关注/取消关注功能,由用户模块提供)
- 个人主页文档
- 笔记详情页面文档

## 一、赞和收藏通知页

### 用户操作
1. 从消息主页点击"赞和收藏"分类
2. **触发接口** → 获取赞和收藏通知列表
3. 下拉刷新 → 重新加载数据
4. 上拉加载 → 加载更多数据
5. 点击通知项 → 跳转到对应内容详情页

### 接口定义

#### 接口1:获取赞和收藏通知列表
**接口:** `GET /api/message/notifications/likes`

**触发时机:** 进入赞和收藏页面时 / 下拉刷新 / 上拉加载

**请求参数:**
```typescript
{
  page: number;           // 页码,从1开始
  pageSize: number;       // 每页数量,默认20
  lastNotificationId?: string; // 最后一条通知ID,用于分页
}
```

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: {
    total: number;        // 通知总数
    hasMore: boolean;     // 是否有更多数据
    list: Array<{
      notificationId: string;   // 通知ID
      userId: string;           // 操作用户ID
      nickname: string;         // 操作用户昵称
      avatar: string;           // 操作用户头像URL
      actionType: 'like' | 'collect'; // 操作类型:点赞/收藏
      targetType: 'note' | 'comment'; // 目标类型:笔记/评论
      targetId: string;         // 目标内容ID
      targetPreview: string;    // 目标内容预览文字
      targetImage?: string;     // 目标内容缩略图(笔记封面)
      timestamp: string;        // 通知时间,ISO格式
      isRead: boolean;          // 是否已读
    }>;
  }
}
```

#### 接口2:标记通知已读
**接口:** `PUT /api/message/notifications/read`

**触发时机:** 进入赞和收藏页面时自动触发

**请求参数:**
```typescript
{
  notificationType: 'likes'; // 通知类型
}
```

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: null
}
```

## 三、粉丝通知页

### 用户操作
1. 从消息主页点击"粉丝"分类
2. **触发接口** → 获取粉丝通知列表
3. 下拉刷新 / 上拉加载
4. 点击"关注"按钮 → 关注用户
5. 点击"互相关注"按钮 → 取消关注
6. 点击用户头像/昵称 → 跳转到用户主页

### 接口定义

#### 接口1:获取粉丝通知列表
**接口:** `GET /api/message/notifications/followers`

**触发时机:** 进入粉丝页面时 / 下拉刷新 / 上拉加载

**请求参数:**
```typescript
{
  page: number;
  pageSize: number;
  lastNotificationId?: string;
}
```

**响应数据:**
```typescript
{
  code: number;
  message: string;
  data: {
    total: number;
    hasMore: boolean;
    list: Array<{
      notificationId: string;
      userId: string;           // 粉丝用户ID
      nickname: string;         // 粉丝昵称
      avatar: string;           // 粉丝头像
      bio?: string;             // 粉丝个性签名
      timestamp: string;        // 关注时间
      isRead: boolean;
      followStatus: 'not_following' | 'following' | 'mutual'; // 关注状态
      // not_following: 未关注
      // following: 已关注(单向)
      // mutual: 互相关注
    }>;
  }
}
```

**说明:** followStatus字段表示当前用户与该粉丝的关注关系状态

#### 接口2:标记粉丝通知已读
**接口:** `PUT /api/message/notifications/read`

**触发时机:** 进入粉丝页面时自动触发

**请求参数:**

- 防重复点击:

  - 防止连续点击造成重复请求

### 前端行为

**点击"关注"按钮:**
1. 按钮进入禁用状态
2. 调用用户模块的关注接口: `POST /api/user/follow`
   - 接口定义详见 **《用户关系文档》**
3. 成功后:
   - 更新按钮状态为"互相关注"或"已关注"
   - 显示提示:"关注成功"
   - 更新本地followStatus状态
4. 失败后:
   - 显示错误提示
   - 恢复按钮状态

**点击"互相关注"/"已关注"按钮:**
1. 弹出确认对话框:"确定取消关注吗?"
   - 按钮1:"取消" - 关闭对话框
   - 按钮2:"确定" - 执行取消关注
2. 点击确定:
   - 调用用户模块的取消关注接口: `DELETE /api/user/follow/{userId}`
   - 接口定义详见 **《用户关系文档》**
   - 成功后:
     - 更新按钮状态为"关注"
     - 显示提示:"已取消关注"
     - 更新followStatus为'not_following'
   - 失败后:
     - 显示错误提示

**点击头像/昵称:**
- 跳转到用户主页
- 传递userId参数

## 通用验证规则

### 登录状态
- 所有接口都需要登录状态
- 未登录时跳转至登录页

### 分页参数
- page:从1开始,必须 ≥ 1
- pageSize:默认20,范围10-50

### 时间格式化
```typescript
const formatNotificationTime = (timestamp: string) => {
  const now = new Date();
  const notificationTime = new Date(timestamp);
  const diff = now.getTime() - notificationTime.getTime();

  // 1小时内
  if (diff < 60 * 60 * 1000) {
    const minutes = Math.floor(diff / (60 * 1000));
    return `${minutes}分钟前`;
  }

  // 今天
  if (now.toDateString() === notificationTime.toDateString()) {
    const hours = notificationTime.getHours().toString().padStart(2, '0');
    const minutes = notificationTime.getMinutes().toString().padStart(2, '0');
    return `${hours}:${minutes}`;
  }

  // 昨天
  const yesterday = new Date(now);
  yesterday.setDate(yesterday.getDate() - 1);
  if (yesterday.toDateString() === notificationTime.toDateString()) {
    return '昨天';
  }

  // 更早
  const month = (notificationTime.getMonth() + 1).toString().padStart(2, '0');
  const day = notificationTime.getDate().toString().padStart(2, '0');
  return `${month}-${day}`;
}
```

## 数据传递

### 跨页面传递

**从消息主页 → 通知子页:**
- 传递:notificationType ('likes' / 'comments' / 'followers' / 'system')
- 方式:路由参数

**从通知子页 → 内容详情页:**
- 赞和收藏 → 笔记详情/评论详情:
  - 传递:targetId, targetType
- 评论 → 笔记详情/评论详情:
  - 传递:targetId, targetType, commentId
- 粉丝 → 用户主页:
  - 传递:userId
- 系统通知 → WebView/应用页面:
  - 传递:linkUrl, linkType

### 状态同步
操作成功后需要同步到:
- 消息主页Tab未读角标(清零)
- 通知列表本地缓存
- 全局消息数

## 异常处理

| 场景 | 前端处理方式 |
|-----|------------|
| 权限不足(403) | 显示:"无权限访问",返回上一页 |
| 资源不存在(404) | 显示:"内容不存在或已删除" |
| 参数错误(400) | 显示:"请求参数错误" |
| 网络错误 | 显示:"网络异常,请检查网络设置" + 重试按钮 |
| 未登录(401) | 跳转登录页 |
| 服务器错误(500) | 显示:"操作失败,请稍后重试" |
| 关注失败 | 显示:"关注失败,请稍后重试",恢复按钮状态,详见用户关系文档 |
| 取消关注失败 | 显示:"取消关注失败,请稍后重试",恢复按钮状态,详见用户关系文档 |

## 性能优化

### 列表优化
- 使用虚拟滚动(长列表超过50条)
- 图片懒加载(头像、缩略图)
- 分页加载,每页20条

### 缓存策略
- 通知列表缓存5分钟
- 下拉刷新清除缓存
- 本地存储已读状态

### 图片优化
- 头像使用CDN缩略图(48x48 / 56x56)
- 笔记封面使用缩略图(64x64)
- 懒加载可视区域外的图片

### 接口优化
- 防抖处理(下拉刷新间隔1秒)
- 接口取消(页面离开时取消未完成请求)
- 标记已读接口防重复调用
